Induction on Natural Numbers

Introduction

The induction principle comes from a generalization of a dependent function that makes recursion on natural numbers. We first define what is a natural number and later, we show how to define functions by means of the recursor and induction principles.

The universe will be denoted by 𝒰:

𝒰 = Set

If we follow an algorithmic procedure to define natural numbers, we will encounter two rules to construct them, one rule to get the zero and another rule for the rest, that we call it, suc for successor.

data ℕ : 𝒰 where
  zero : ℕ
  suc  : ℕ → ℕ

For convenience, we use the following Agda pragma to write down numbers in decimal notation, otherwise, we should write the number down using only the constructors.

{-# BUILTIN NATURAL ℕ #-}

And this is what we got.

bigNumber : ℕ
bigNumber = 123456789  -- instead of suc (suc ( ... (suc ..)..

Recursion principle

The principle of primitive recursion for natural numbers establishes that to create a non-dependent outgoing function with natural numbers as its domain, we just need to tell it the type in the codomain, and the function behavior on the ℕ constructors. Which means, says what return in the case when the number is zero and when the number was built using suc constructor.

In Agda, the recℕ also called recursor for natural number has been defined by pattern-matching as follows.

recℕ
  : (C : 𝒰)
  → C
  → (ℕ → C → C)
  -------------
  → ℕ → C

With the following equations:

recℕ C c₀ cₛ zero    = c₀
recℕ C c₀ cₛ (suc n) = cₛ n (recℕ C c₀ cₛ n)

Examples

Add

add : ℕ → ℕ → ℕ
add = recℕ (ℕ → ℕ) (λ m → m) (λ n g m → suc (g m))

_+_ = add
infix 6 _+_

Instead of using the following definition:

add₂ : ℕ → ℕ → ℕ
add₂ zero    m = m
add₂ (suc n) m = suc (add₂ n m)

Double

double : ℕ → ℕ
double = recℕ ℕ 0 (λ n y → suc (suc y))

Instead of:

double₂ : ℕ → ℕ
double₂ zero = zero
double₂ n    = suc (suc n)

For testing purposes, let’s import from the equaility definition type (_≡_) and its rule (refl) from the std-lib library.

open import Relation.Binary.PropositionalEquality using (refl; _≡_; sym)

2+5 : add 2 5 ≡ 7
2+5 = refl

25+25 : add 25 25 ≡ 50
25+25 = refl

It’s time to unpacking the the definition of add:

  • By Currying, the binary function add can be seen as a function that returns a unary function fixing the first argument. Thus, the domain for the recℕ, C is ℕ → ℕ (a unary funciton).
  add : ℕ → (ℕ → ℕ)
  add n : ℕ → ℕ
  • When the first argument is zero, we just return the second argument, that is, add 0 is the identity function. Thus c₀ is (λ m → m).
  add zero m = m
  • Question: why ((λ n g m → suc (g m)))?

Multiplication

_*_ : ℕ → ℕ → ℕ
_*_ = recℕ (ℕ → ℕ) (λ m → zero) (λ n g m → add m (g m))

With the binding for this operation more tighly than (+)

infix 7 _*_
m₁ : 2 * 0 ≡ 0
m₁ = refl

m₂ : 2 * 3 ≡ 6
m₂ = refl

m₃ : 10 * 3 ≡ 30
m₃ = refl

Induction principle

The induction here is a generalization of the principle of recursion. In first-order we can write the induction schema or the principle of mathematical induction.

In particular, a property of natural numbers is represented by a family of types P : ℕ → 𝒰. From this point of view, the above induction principle says that if we can prove P(0), and if for any n we can prove P(succ(n)) assuming P(n), then we have P(n) for all n. (Pp. 50-51 in (Univalent Foundations Program, 2013))

By using a dependent function to obtain its version in type theory we have the following

indℕ
  : ∀ {C : ℕ → 𝒰}
  → C zero
  → (∀ (n : ℕ) → C n → C (suc n))
  -------------------
  → ∀ (n : ℕ) → C n

with the defining equations

indℕ c₀ cₛ zero    = c₀
indℕ c₀ cₛ (suc n) = cₛ n (indℕ c₀ cₛ n)

Now, we have the power of induction to prove some properties.

Congruence

suc-cong
  : ∀ {n m : ℕ}
  -----------------------
  → n ≡ m → suc n ≡ suc m

suc-cong refl = refl

As we can see in the type of suc-cong we used implicit arguments for the numbers n and m. That’s pretty convenient to get some help by letting infer Agda about those implicit argument.

Using congruence we can now prove that both definitions above for the add function are indeed equal using ι-,β- reductions:

add≡add₂
  : ∀ (n m : ℕ)
  --------------------
  → add n m ≡ add₂ n m

add≡add₂ zero    m = refl
add≡add₂ (suc n) m = suc-cong (add≡add₂ n m)

Associativity

assoc
  : (i j k : ℕ)
  ---------------------------
  → i + (j + k) ≡ (i + j) + k

To prove this property by induction we need first to provide the term assoc₀, that is the base case and then build an inhabitant of the induction hypothesis.

assoc₀
  : ∀ (j k : ℕ)
  ---------------------------
  → 0 + (j + k) ≡ (0 + j) + k

assoc₀ j k = refl
assoc₁
  : ∀ (i : ℕ)
  → ((j k : ℕ) → i + (j + k) ≡ (i + j) + k)
  ---------------------------------------------------
  → (j k : ℕ) → (suc i) + (j + k) ≡ ((suc i) + j) + k

assoc₁ i p j₁ k₁ = suc-cong (p j₁ k₁)

Then, by indℕ:

assoc = indℕ assoc₀ assoc₁

Commutativity

+-comm₀
  : ∀ (m : ℕ)
  ---------------
  → 0 + m ≡ m + 0

+-comm₀ = indℕ refl (λ n indHyp → suc-cong indHyp)
+-identity
  : ∀ (n : ℕ)
  --------------
  → n + zero ≡ n

+-identity = indℕ refl (λ n indHyp → suc-cong indHyp)
+-suc
  : ∀ m n
  -------------------------
  → m + suc n ≡ suc (m + n)

+-suc zero    n = refl
+-suc (suc m) n = suc-cong (+-suc m n)

Let’s define the transitivity and symmetric property of the equality.

trans
  : ∀ {m n p : ℕ}
  → m ≡ n → n ≡ p
  ---------------
  → m ≡ p

trans refl refl = refl
≡sym
  : ∀ {m n p : ℕ}
  → m ≡ n → n ≡ m

≡sym refl = refl
+-comm
  : ∀ (m n : ℕ)
  ---------------
  → m + n ≡ n + m

+-comm =
  indℕ
    sproof₁
    sproof₂
  where
    sproof₁
      : ∀ (n : ℕ)
      ----------------
      → n ≡ (n + zero)

    sproof₁ = indℕ refl (λ n n≡n+zero → suc-cong n≡n+zero)

    sproof₂
      : ∀ (n : ℕ)
      → ((m : ℕ) → (n + m) ≡ (m + n))
      ---------------------------------------
      → ((m : ℕ) → suc (n + m) ≡ (m + suc n))

    sproof₂ n hyp₁ =
      indℕ
        (suc-cong
          (hyp₁ zero) )
        (λ m hyp₂ →
            suc-cong
              (trans
                  (hyp₁ (suc m))
              (trans
                  (suc-cong
                      (sym (hyp₁ m)))
                  hyp₂)))

Alternative induction

Assuming the ordinary induction principle (i.e., indℕ) derives the transfinite induction principle.
For a unary predicate $$P : \mathbb{N} \to \mathcal{U}$$, if

$$ \prod\limits_{n : \mathbb{N}} \left ( \prod\limits_{k : \mathbb{N}} (k < n \to P(k)) \to P(n) \right) $$

then for all $$n : \mathbb{N}$$, $$P(n)$$.

To solve this problem, we need to define a type for the less than relationship (_<_) between natural numbers but we also have to define the disjunction to make a case analysis that we need for our proof. Let’s see. You may skip this first part.

module ℕ-transInd (P : ℕ → 𝒰) where

  data _<_ : ℕ → ℕ → 𝒰 where
    z<s : ∀ {n : ℕ}   → zero < suc n
    s<s : ∀ {m n : ℕ} → m < n → suc m < suc n

  data _⊎_ : 𝒰 → 𝒰 → 𝒰 where
    inj₁ : ∀ {A B : 𝒰} → A → A ⊎ B
    inj₂ : ∀ {A B : 𝒰} → B → A ⊎ B

  ⊎-elim
    : ∀ {A B C : 𝒰}
    → (A → C)
    → (B → C)
    -------------
    → (A ⊎ B → C)

  ⊎-elim f g (inj₁ x) = f x
  ⊎-elim f g (inj₂ y) = g y

  subst : {k n : ℕ} → k ≡ n → P k → P n
  subst refl pk = pk

  split-k<sucn
    : ∀ {k : ℕ} {n : ℕ}
    → k < suc n
    -------------------
    → (k < n) ⊎ (k ≡ n)

  split-k<sucn {zero}  {zero}  k<sucn = inj₂ refl
  split-k<sucn {zero}  {suc n} k<sucn = inj₁ z<s
  split-k<sucn {suc k} {zero}  (s<s ())
  split-k<sucn {suc k} {suc n} (s<s k<sucn) =
    ⊎-elim
      (λ k<n → inj₁ (s<s k<n))
      (λ k≡n → inj₂ (suc-cong k≡n))
      (split-k<sucn k<sucn)
Proof.
We use induction to get an inhabitant of the $$G$$ proposition. The induction was using pattern matching on $$n$$ in Agda. At the end, we use the hypothesis with this inhabitant of $$G$$. {: .equation } $$ G : \prod\limits_{(n : \mathbb{N})}\ \left(\prod\limits_{(k : \mathbb{N})}\ ((k < n) \to P (k))\right) $$ where $$P : \mathbb{N} \to \mathcal{U}$$. 

-- proof:
  indℕ⇒transFindℕ
    : (hyp : (n : ℕ) → ((k : ℕ) → (k < n) → P k) → P n)
    → ((n : ℕ) → P n)

  indℕ⇒transFindℕ hyp zero    = hyp zero (λ k → λ ())
  indℕ⇒transFindℕ hyp (suc n) = hyp (suc n) (G (suc n))
    where
      G : ∀ (n : ℕ) → ((k : ℕ) → (k < n) → P k)
      G zero    k () -- impossible case
      G (suc n) k k<n+1 =
        ⊎-elim
          -- 1. when k < n
          (λ k<n → G n k k<n)
          -- 2. when k ≡ n
          (λ k≡n → subst (sym k≡n) (hyp n (G n)))
          -- eliminiting two cases: (k < n) ⊎ (k ≡ n)
          (split-k<sucn k<n+1)

Summary

The recursor recℕ is the no-dependent version of indℕ function that allows us to define functions f : ℕ → C where C : 𝒰 by defining two equations:

  • f(0) ≡ c₀ for c₀ : C

  • f(suc n) ≡ cₛ(n, f(n)) for cₛ : ℕ → C → C
