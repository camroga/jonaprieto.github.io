Path Algebra in HoTT

In Univalence we have a different interpretation of type theory (Univalent Foundations Program, 2013). We replace the set-theoretical notion of sets for types and we use the topological space notion instead. The judgement for a type is now the point in the topological space . We also include the identity type as a primary type. Changing the notation of this type about a proof of the equality to a path space. This path space comprehends all paths with as the starting point and as the end point. The inhabitant of this type is called a path.

Prerequisites

To formalize some of the HoTT concepts, let’s use Agda (Team, 2015). To work with the identity type, we (_≡_) defined in the Agda standard library but using the following pragma to make our code compatible with HoTT.

{-# OPTIONS --without-K #-}
open import Relation.Binary.PropositionalEquality using (_≡_; refl)

Path Induction

The elimination principle for the identity type is the path induction. It allows us to define an outgoing function from the identity type to a dependent type as we see in the pi definition. It worths to mention this principle is also called J.

pi
  : ∀ {i j} {A : Set i}
  → (C : (x y : A) → x ≡ y → Set j)
  → (∀ (x : A) → C x x refl)
  → ∀ (x y : A) (p : x ≡ y) → C x y p
pi {A} C c x .x refl = c x

Path Concatenation

To join two paths when one ends where the other starts, we define the concatenation operator between paths denoted by the symbol (_·_). Let’s see its picture.

infixr 20 _·_
_·_ : ∀ {A : Set} {x y z : A}
    → (p : x ≡ y)
    → (q : y ≡ z)
    ------------
    → x ≡ z

_·_ {A} {x} {y} {z} p q = D₁ x y p z q
  where
    D₂ : (x z : A) (q : x ≡ z) → x ≡ z
    D₂ = pi (λ x z q → x ≡ z) (λ x → refl)

    D₁ : (x y : A) → (x ≡ y) → ((z : A) → (q : y ≡ z) → x ≡ z)
    D₁ = pi (λ x y p → ((z : A) → (q : y ≡ z) → x ≡ z)) (λ x → D₂ x)

To make the above code shorter in Agda, we could have defined the function by pattern-matching. Nonetheless, the idea here was use path induction — the pi function— entirely.

Path Inverse

The path inverse for a given path p : a = b is denoted by p⁻¹ : b = a. This path only change the original direction of the path p. Let’s see it.

infixl 20 _⁻¹
_⁻¹ : ∀ {A : Set} {x y : A} → (p : x ≡ y) → y ≡ x
_⁻¹ {A}{x}{y} p =
  pi (λ x y p → y ≡ x)
     (λ x → refl {x = x})
     x y p

As you can see, path inversion is just the symmetric property for this identity type. Now let’s see some algebra.

Transport

transp : ∀ {A : Set}{x x' : A}
      → (B : A → Set) → (y : B x) → (u : x ≡ x') → B x'
transp B y refl  = y

We should also say, we’re already familiar with its type since it shows up as the susbstitution of equals in type theory. Actually, in the Agda standard library, we found subst that follows from the same principle:

open import Relation.Binary.PropositionalEquality using (subst)

trans' : ∀ {A : Set}{x x' : A}
      → (B : A → Set) → (y : B x) → (u : x ≡ x') → B x'
trans' B y u = subst B u y
