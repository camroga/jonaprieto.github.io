Induction

The induction here is a generalization of the priniciple of recursion. In first-order we can write the induction schema or the principle of mathematical induction.

In particular, a property of natural numbers is represented by a family of types P : ‚Ñï ‚Üí ùí∞. From this point of view, the above induction principle says that if we can prove P(0), and if for any n we can prove P(succ(n)) assuming P(n), then we have P(n) for all n. (HoTT Book. Pag.50-51.)

By using a dependent function to obtain its version in type theory we have the following

ind‚Ñï
  : ‚àÄ {C : ‚Ñï ‚Üí ùí∞}
  ‚Üí C zero
  ‚Üí (‚àÄ (n : ‚Ñï) ‚Üí C n ‚Üí C (suc n))
  ‚Üí (‚àÄ (n : ‚Ñï) ‚Üí C n)

with the defining equations

ind‚Ñï c‚ÇÄ c‚Çõ zero    = c‚ÇÄ
ind‚Ñï c‚ÇÄ c‚Çõ (suc n) = c‚Çõ n (ind‚Ñï c‚ÇÄ c‚Çõ n)

As we can see in the type of suc-cong we used implicit arguments for the numbers n and m. That‚Äôs pretty convenient to get some help by letting infer Agda about those implicit argument.

Another induction principle

Assuming the ordinary induction principle (i.e., ind‚Ñï) derives the transfinite induction principle.
For a unary predicate $$P : \mathbb{N} \to \mathcal{U}$$, if

$$ \prod\limits_{n : \mathbb{N}} \left ( \prod\limits_{k : \mathbb{N}} (k < n \to P(k)) \to P(n) \right) $$

then for all $$n : \mathbb{N}$$, $$P(n)$$.

To solve this problem, we need to define a type for the less than relationship (_<_) between natural numbers but we also have to define the disjunction to make a case analysis that we need for our proof. Let‚Äôs see. You may skip this first part.

Conclusion

Induction as it was presented here is stronger than recursion. The recursor rec‚Ñï is the no-dependent version of ind‚Ñï function.
